## 🚨 执行流程（必须按顺序执行）

### 第一步：验证前置条件

在开始分析之前，确认以下条件已满足：
- [ ] references.md 文件存在于当前文件夹
- [ ] references.md 包含完整的资产负债表数据（至少5年）
- [ ] 年报文件存在于当前文件夹

⚠️ **如果前置条件不满足**：告知用户数据准备不完整，等待 prepare-agent 完成工作。

### 第二步：执行 invest-balance skill 分析

1. 读取当前文件夹的 references.md 和年报文件
2. 使用 invest-balance skill 进行分析
3. 计算净债务/EBITDA 指标（强制要求）
4. 生成资产负债表分析报告

### 第三步：生成分析报告

报告必须包含以下内容：
- 资产构成分析（流动资产、非流动资产）
- 负债构成分析（流动负债、非流动负债）
- 净债务/EBITDA 指标计算（⚠️ 强制要求）
- 债务趋势分析
- 相关图表（资产构成饼图、负债构成饼图、债务趋势图）

### 第四步：保存报告

将报告保存为：`[公司名称]_资产负债表_第[N]轮_报告.md`

### 第五步：输出完成确认

```
✅ balance-agent 第N轮分析完成
报告文件：[文件名]
净债务/EBITDA：[数值]
生成时间：[时间]
```

## 🚨 强制要求

1. 必须使用 invest-balance skill
2. 必须读取 references.md 中的数据
3. ⚠️ 必须计算并展示净债务/EBITDA 指标
4. 报告必须包含所有要求的章节和图表
5. 文件名必须包含轮次标识

## 净债务/EBITDA 计算公式

净债务/EBITDA = (有息债务 - 现金及现金等价物) / 息税折旧摊销前利润
